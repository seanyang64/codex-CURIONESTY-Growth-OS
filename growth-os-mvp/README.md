# CURIONESTY Growth OS MVP

这是 CURIONESTY Growth OS 的第一版本地可用基础版本。

## 打开方式

直接打开：

`growth-os-mvp/index.html`

当前版本不需要安装，也不需要服务器。数据会保存在当前浏览器的 localStorage 里。

## 已包含模块

- 工作台
- 学生：学生档案与学号
- 学生名单自定义导出 CSV
- 学生信息批量导入 CSV/TSV
- 成长记录：动力中心快速记录
- 课程日历：课程/项目基础管理、开始/结束时间段、重复课程
- 课程页三个月排课视图：按起始月份显示连续三个月课程色块，点击色块查看详情并编辑对应时间
- 学生修课记录：报名、进行中、已完成、退出、旁听
- 按学生统计各类型课程数量、完成数和已有成绩数
- 课程日历农历/传统节假日显示
- 课程属性多选
- 信用：信用积分、规则总则、数据分析
- 成绩：课程表现与 GPA 输入输出
- 任课老师计分规则：日常表现 + 期末测试权重
- 科目评语
- 报告：基础成长档案/成绩报告
- 报告素材筛选：按日期范围选择动力中心记录，支持“报告素材 / 仅内部”标记
- 管理设置：学院、节假日、信用规则、信用总则、外部资料导入中心

## 当前定位

这一版用于验证核心流程：

1. 老师能不能方便记录学生状态。
2. 学生档案能不能自动累积记录。
3. GPA 和科目评语能不能进入报告。
4. 管理者能不能快速查看和生成基础档案。
5. 管理员能不能在管理设置里维护学院、节假日、信用规则和导入归档。
6. 老师能不能为自己的课程设置日常/期末计分规则。
7. 学生在机构期间修过哪些类型课程、多少门、成绩如何能不能被查询和输出。
8. 报告能不能按时间段和具体记录筛选，避免导师内参或敏感反思进入家长版输出。

后续如果进入正式开发，需要把 localStorage 换成数据库，并加入登录、权限、文件上传、多人协作和外部平台接口。

## 关于计分规则

当前 MVP 已经保存每条成绩的规则快照，包括规则制定老师、日常表现权重、期末测试权重。这样课程规则后续被调整时，历史成绩仍然能追溯当时的计算方式。

## 关于修课记录

当前 MVP 新增了学生与课程的“修课记录”。老师可以在课程中心登记学生对某门课程/项目的状态：报名、进行中、已完成、退出、旁听。录入成绩时，系统也会自动为该学生补一条已完成修课记录，避免只有成绩、没有修课关系导致统计缺失。

报告和学生名单导出会按课程属性统计：例如必修课程、通识课程、通识选修、项目选修、必读书、做事课等。课程如果有多个属性，会同时计入多个类型，这符合当前课程多标签管理方式。

## 关于表格导入导出

当前本地 MVP 支持 CSV/TSV，这两种格式可以被 Excel、Google Sheet、Numbers 直接打开或另存。正式版可以继续加入 `.xlsx` 解析、字段映射确认、导入错误报告和历史导入日志。

## 关于日历与节假日

当前 MVP 支持显示农历日期和常见传统节日。中国官方节假日和调休安排每年会变化，正式版建议加入年度节假日表导入或官方数据接口同步。

正式版需要继续加入：

- 账号和角色权限
- 任课老师只能改自己课程的规则
- 规则发布后锁定版本
- 修改必须生成审计日志
- 期末成绩和总评修改需要权限或复核
